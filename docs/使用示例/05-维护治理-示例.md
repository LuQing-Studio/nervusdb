# 示例 05 · 维护与治理

## 目标

- 建立日常巡检、压实、GC、修复的脚本化流程
- 演示如何将 CLI 输出写入日志并分析

## 核心脚本

```bash
#!/usr/bin/env bash
set -euo pipefail
DB=$1
LOG_DIR=/var/log/synapsedb
mkdir -p "$LOG_DIR"

synapsedb stats "$DB" --summary --json > "$LOG_DIR/$(date +%F)-stats.json"
synapsedb auto-compact "$DB" --mode=incremental --hot-threshold=1.1 --max-primary=5 --auto-gc
synapsedb txids "$DB" --since=60 --json > "$LOG_DIR/$(date +%F)-txids.json"
```

## 关键命令

```bash
synapsedb check demo.synapsedb --strict
synapsedb repair demo.synapsedb --fast
synapsedb hot demo.synapsedb --top=10
synapsedb gc demo.synapsedb --respect-readers
```

## 日志解析

```bash
jq '.orders.SPO.multiPagePrimaries' logs/2025-09-28-stats.json
```

## 自动化建议

- 使用 `cron` 每小时执行 auto-compact
- 使用 `systemd` timer 保证重启后仍能运行
- 将 JSON 日志推送到 ELK / Loki 统一分析
